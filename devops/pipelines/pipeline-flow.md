## Branch Strategy

```
┌─────────────────────────────────────────────────────────────────┐
│                     BRANCH STRATEGY                            │
└─────────────────────────────────────────────────────────────────┘

main (production)
  │
  ├── feature/* (development)
  │     │
  │     └── Pull Request → Code Review → Merge to main
  │
  ├── hotfix/* (emergency fixes)
  │     │
  │     └── Direct merge to main → Deploy
  │
  └── release/* (release branches)
        │
        └── Merge to main → Tag → Deploy
```

## Key Decision Points

| Stage | Decision | Action |
|-------|----------|--------|
| Test | Tests pass | Continue to next stage |
| Test | Tests fail | Send notification, stop pipeline |
| Quality Gate | Gate passes | Continue to next stage |
| Quality Gate | Gate fails | Send notification, stop pipeline |
| Staging | Health check pass | Deploy to production |
| Staging | Health check fail | Rollback to previous version |
